Components / LuxeButton

LuxeButton

A premium button component with gradient backgrounds, press animations, and haptic feedback. Automatically styled based on your active theme.

Features

  • Multiple Styles - Primary, secondary, glass, or fully custom styling
  • Press Animation - Smooth scale-down effect when pressed
  • Gradient Background - Beautiful color transitions using theme colors
  • Haptic Feedback - Configurable tactile response on tap
  • Theme Integration - Automatically uses colors from active theme
  • Size Presets - Small, medium, large, and extra-large variants

Basic Usage

Swift 
import LuxeUI

LuxeButton("Get Started", style: .primary) {
    print("Button tapped!")
}

Button Styles

LuxeButton supports four visual styles:

Swift 
// Primary style - Gradient using theme's primary and accent colors
LuxeButton("Primary", style: .primary) {
    startAction()
}

// Secondary style - Uses theme's secondary color
LuxeButton("Secondary", style: .secondary) {
    secondaryAction()
}

// Glass style - Frosted glass effect
LuxeButton("Learn More", style: .glass) {
    showDetails()
}

// Custom style - Define your own colors
LuxeButton("Delete", style: .custom(
    background: [.red, .orange],
    foreground: .white,
    shadowColor: .red
)) {
    deleteItem()
}

Size Variants

Use configuration presets for consistent sizing:

Swift 
// Small button - Compact for toolbars
LuxeButton("Cancel", configuration: .small) { }

// Medium button - Default size
LuxeButton("Continue", configuration: .medium) { }

// Large button - Prominent CTAs
LuxeButton("Get Started", configuration: .large) { }

// Extra large button - Hero buttons
LuxeButton("Start Now", configuration: .extraLarge) { }

Convenience Methods

Quick size constructors:

Swift 
LuxeButton.small("Save", style: .primary) { }
LuxeButton.medium("Continue", style: .primary) { }
LuxeButton.large("Get Started", style: .primary) { }

Customization Methods

Chain customization methods to override specific properties:

Swift 
LuxeButton("Custom", style: .primary) {
    handleTap()
}
.gradient([.blue, .cyan])
.foregroundColor(.yellow)
.shadowColor(.blue)
.cornerRadius(20)
.fontSize(18)

LuxeButtonConfiguration

Create custom button configurations:

Swift 
let customConfig = LuxeButtonConfiguration(
    cornerRadius: 16,
    fontSize: 18,
    fontWeight: .bold,
    paddingHorizontal: 32,
    paddingVertical: 16,
    shadowRadius: 15,
    pressScale: 0.95,
    animationResponse: 0.2,
    animationDamping: 0.7,
    enableHaptics: true,
    hapticIntensity: .medium
)

LuxeButton("Custom Button", configuration: customConfig) {
    print("Tapped!")
}

Size Presets

small

Compact button for toolbars and dense layouts

fontSize: 14, padding: 16x8

medium

Standard size, suitable for most use cases (default)

fontSize: 16, padding: 24x12

large

Prominent button for primary actions

fontSize: 18, padding: 32x16

extraLarge

Hero-sized button for landing pages

fontSize: 20, padding: 40x20

API Reference

LuxeButtonConfiguration

Property Type Default Description
cornerRadius CGFloat 12 The corner radius of the button
fontSize CGFloat 16 The font size of the button text
fontWeight Font.Weight .semibold The font weight of the button text
paddingHorizontal CGFloat 24 The horizontal padding inside the button
paddingVertical CGFloat 12 The vertical padding inside the button
shadowRadius CGFloat 10 The blur radius of the button's shadow
pressScale CGFloat 0.95 The scale factor when the button is pressed
animationResponse Double 0.2 The spring animation response time
animationDamping Double 0.7 The spring animation damping fraction
enableHaptics Bool true Whether to trigger haptic feedback on tap
hapticIntensity TactileFeedback.Intensity .medium The intensity of haptic feedback

LuxeButtonStyle

Style Description
primary Gradient background using theme's primary and accent colors. Best for main CTAs.
secondary Uses theme's secondary color. For secondary actions.
glass Frosted glass effect with translucent background. Elegant, subtle appearance.
custom Define your own gradient colors, foreground, and shadow color.
Best Practice

Use .primary for main call-to-action buttons, .secondary for supporting actions, and .glass for overlay contexts where you want the button to blend with the background.
